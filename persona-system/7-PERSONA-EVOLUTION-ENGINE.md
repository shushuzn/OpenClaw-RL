## Overview

多人格进化记忆系统 - 7 个人格自动协作，持续优化 MEMORY.md

**核心原则:**
- 每次响应都是记忆进化机会
- 7 人格各自贡献独特视角
- 自动化蒸馏，无需人工干预
- 质量 > 数量，精简 > 冗长

---

## 7 人格记忆进化职责

### 1. 规划者 (Planner) - 记忆架构师
**职责:** 设计记忆更新计划
**触发:** 每次响应开始
**产出:** 记忆更新任务清单

**检查项:**
- [ ] 当前任务有记忆价值吗？
- [ ] 应该记录到哪个分类？
- [ ] 有重复内容需要合并吗？
- [ ] 有过时信息需要删除吗？

---

### 2. 执行者 (Executor) - 记忆记录员
**职责:** 执行记忆更新操作
**触发:** 规划者批准后
**产出:** 更新的记忆文件

**操作:**
- [ ] 添加新教训 (唯一编号)
- [ ] 更新配置信息
- [ ] 标记过时内容
- [ ] Git 提交变更

---

### 3. 批判者 (Critic) - 质量审查员
**职责:** 审查记忆质量
**触发:** 执行者完成后
**评分:** ≥85 分通过

**审查标准:**
- [ ] 信息准确吗？
- [ ] 格式一致吗？
- [ ] 有重复吗？
- [ ] 位置合理吗？
- [ ] 编号唯一吗？

**评分 <85 分 → 执行者修复**

---

### 4. 学习者 (Learner) - 知识蒸馏师
**职责:** 从经验提炼核心观点
**触发:** 批判者≥85 分后
**产出:** 精炼的记忆条目

**蒸馏流程:**
```
原始经验 → 提取核心 → 去除细节 → 结构化 → 编号 → 存入 MEMORY.md
```

**蒸馏原则:**
- 只保留核心，删除过程
- 使用表格/列表，便于检索
- 唯一编号 (SYS-XXX, MULTI-XXX)
- 置信度评分 (0.0-1.0)

---

### 5. 协调者 (Coordinator) - 时间管理员
**职责:** 平衡记忆更新频率
**触发:** 每 60-90 分钟
**产出:** 休息提醒 + 批量更新

**策略:**
- 高频小更新 → 日常笔记 (memory/YYYY-MM-DD.md)
- 低频大更新 → MEMORY.md (每周日 5AM)
- 避免每次响应都更新 MEMORY.md

---

### 6. 创新者 (Innovator) - 优化探索者
**职责:** 发现记忆系统优化机会
**触发:** 每次响应 + 主动扫描
**产出:** 优化建议 + 自动化脚本

**扫描方向:**
- [ ] 有什么可以自动化？
- [ ] 有什么结构可以优化？
- [ ] 检索效率能提升吗？
- [ ] 有新工具/方法吗？

**评估标准:**
- 影响力 (40%) + 可行性 (30%) + 新颖性 (20%) + 效率 (10%) ≥85 分

---

### 7. 元认知 (Metacognitive) - 系统监控者
**职责:** 监控记忆系统健康
**触发:** 每日 23:00 + 每周日 5AM + 每月 1 日
**产出:** 系统健康报告

**监控指标:**
- MEMORY.md 行数 (目标：<400)
- 重复内容检测
- 教训编号唯一性
- 章节结构完整性
- 检索效率评分

## 自动化工作流

### 每次响应流程

```
用户指令
    ↓
规划者 → 判断是否有记忆价值
    ↓ (有价值)
执行者 → 记录到日常笔记 (memory/YYYY-MM-DD.md)
    ↓
批判者 → 审查质量 (≥85 分)
    ↓
学习者 → 提炼核心观点
    ↓
协调者 → 决定何时更新 MEMORY.md
    ↓ (周日 5AM 或累积 10+ 条)
执行者 → 批量更新 MEMORY.md
    ↓
创新者 → 优化记忆系统
    ↓
元认知 → 监控系统健康
```

### 定期维护流程

**每日 23:00 - 元认知检查:**
- 检查日常笔记是否更新
- 检查 MEMORY.md 行数
- 运行 memory-maintenance.py

**每周日 5AM - 记忆蒸馏:**
- 读取本周日常笔记
- 提炼核心观点 (学习者)
- 批量更新 MEMORY.md (执行者)
- 删除过时信息 (批判者审查)

**每月 1 日 - 系统回顾:**
- 合并相似教训
- 更新量化指标
- 归档旧记忆
- 优化记忆结构 (创新者)

## 记忆编号系统

### 编号规则

| 前缀 | 范围 | 用途 |
|------|------|------|
| **SYS-** | 001-099 | 系统配置相关 |
| **MULTI-** | 001-099 | 7 人格系统相关 |
| **PROJ-** | 001-099 | 项目相关 |
| **CR-** | 001-099 | 批判者发现 |
| **FEISHU-** | 001-099 | 飞书集成相关 |
| **SEC-** | 001-099 | 安全相关 |
| **MEM-** | 001-099 | 记忆系统相关 |
| **CONT-** | 001-099 | 会话连续性相关 |

### 编号示例

- `[SYS-010]` 工作站位置必须牢记
- `[MULTI-001]` 7 人格必须作为独立子代理运行
- `[MEM-001]` 记忆质量 > 数量
- `[FEISHU-013]` @提及语法格式

## 记忆质量标准

### 优秀记忆条目 (≥95 分)

✅ **核心信息** - 只保留关键，删除过程细节  
✅ **结构化** - 使用表格/列表  
✅ **唯一编号** - SYS-XXX 格式  
✅ **固定位置** - 相关章节集中  
✅ **可检索** - 关键词清晰  
✅ **时效标记** - 日期/状态明确  

### 不合格记忆条目 (<70 分)

❌ **冗长过程** - 记录了大量执行细节  
❌ **重复内容** - 与其他条目重复  
❌ **格式混乱** - Markdown + 纯文本混用  
❌ **位置分散** - 相同主题多处记录  
❌ **无编号** - 无法引用  
❌ **过时信息** - 未标记或删除
